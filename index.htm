Kajian Teori
Arsitektur Perangkat Lunak (Software Architecture)
Merupakan sekumpulan pernyataan yang menggambarkan perangkat lunak beserta fungsi-fungsi yang tergabung dalam komponen tersebut, termasuk struktur teknis, batasan-batasan, ciri-ciri, serta antarmukanya.

Pada dasarnya, arsitektur disini merupakan rancangan fisik sistem.
Hypertext Transfer Protocol (HTTP) 
- Terletak di application layer pada arsitektur protokol TCP/IP.
- Tidak menentukan bagaimana data diproses tetapi menentukan bagaimana data ditransfer.
- Tidak hanya mentransfer halaman web, tetapi dapat mentransfer semua format data.
- Tidak hanya digunakan oleh web browser.
Application Programming Interface (API)
adalah sebuah protokol yang dimaksudkan untuk digunakan sebagai antarmuka oleh komponen perangkat lunak untuk berkomunikasi antara satu sama lain.
Kenapa menggunakan API ?
- Memungkinkan pengguna mengakses data mereka melalui aplikasi pihak ketiga
- Memungkinkan pengembang aplikasi mobile untuk mengakses situs dengan mengirim dan mengambil data menggunakan HTTP
- Memungkinkan aplikasi untuk mengakses data sendiri melalui panggilan AJAX
Contoh Kategori API
Representational State Transfer (REST)
RESTful API adalah API yang diimplementasikan dengan melalui HTTP dan prinsip-prinsip REST.

Representational State Transfer (REST) mendeﬁnisikan seperangkat prinsip arsitektur dimana penulis dapat merancang layanan Web yang berfokus pada sistem resource, termasuk bagaimana keadaan resource dipanggil dan ditransfer melalui HTTP oleh berbagai macam klien yang ditulis dalam bahasa (pemrograman) yang berbeda.
JavaScript Object Notation (JSON)
adalah format data interchange ringan berbasis teks yang bersifat language-independent.

Perancangan Sistem
- Sistem harus berupa web service yang diimplementasikan sesuai dengan prosedur arsitektur REST API dan mampu memberikan data balasan berupa JSON agar bisa diolah kembali oleh bahasa pemrograman lain dengan mudah. Nantinya JSON ini juga dapat diolah untuk membuat aplikasi berbasis web, desktop hingga aplikasi mobile seperti Android, Windows Phone, iOS, dan lain-lain.

- Sistem harus mampu melakukan autentifikasi request ke web service dengan menggunakan API Key untuk mengenali sistem terintegrasi yang melakukan request.

- Sistem harus mampu memberikan kode status response yang jelas yaitu kode status ketika sukses, ketika terjadi kesalahan di server, maupun ketika terjadi kesalahan dari sisi klien.

- Sistem harus memiliki dokumentasi penggunaan API yang jelas dan mudah dimengerti oleh developer.
Kebutuhan Sistem
Perancangan API
API dirancang dengan konsep REST. Memudahkan klien dalam melakukan request melalui protokol HTTP dengan hanya memanfaatkan URI.
http://{namaDomain}/{subDomain}/{namaFungsi}?{parameter_1}...&{parameter_n}
Dimana:
1. {namaDomain} adalah nama domain letak API berada.
2. {subDomain} adalah sub folder letak API berada.
3. {namaFungsi} adalah nama fungsi yang akan diakses.
4. {parameter_1}...&{parameter_n} adalah parameter yang dikirim.
